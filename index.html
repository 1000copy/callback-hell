# Callback Hell *A guide to writing elegant asynchronous javascript programs* ### What is "*callback hell*"? Asynchronous javascript, or javascript that uses callbacks, is hard to get right intuitively. A lot of code ends up looking like this: getThingFromHardDrive(function(thing) { uploadToServer(thing, function(response) { saveToLogDatabase(response, function(logResponse) { getAnotherThingFromHardDrive(function(thing) { // etc }) }) }) }) See all the instances of `function` and `})`? Eek! This is affectionately known as **callback hell**. In the above code example there are four functions being nested inside one another.
